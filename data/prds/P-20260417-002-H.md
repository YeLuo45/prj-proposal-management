# PRD: android-hello BuildConfig + 多环境

## 1. 概述
为 android-hello 配置多环境构建（dev/staging/prod），支持不同服务器地址、特性开关和构建变体。

## 2. 功能需求

### 2.1 环境配置
- **dev**: 开发环境，本地调试用
- **staging**: 测试环境，QA 验收用
- **prod**: 生产环境，正式发布用

### 2.2 BuildConfig 字段
- `ENV_NAME`: 环境名称（dev/staging/prod）
- `BASE_URL`: API 基础地址（各环境不同）
- `ENABLE_LOGGING`: 是否启用日志
- `APP_VERSION_NAME`: 应用版本名

### 2.3 构建变体
- `devDebug`: 开发调试版本
- `devRelease`: 开发发布版本
- `stagingDebug`: 测试调试版本
- `stagingRelease`: 测试发布版本
- `prodDebug`: 生产调试版本
- `prodRelease`: 生产发布版本

## 3. 技术方案
- productFlavors 配置多环境
- buildConfigField 定义环境变量
- gradle.properties 管理敏感信息

## 4. 验收标准
- [ ] 三个环境配置正确
- [ ] BuildConfig 字段可访问
- [ ] 各变体 APK 可独立构建
- [ ] 切换环境无需修改代码

## 5. 依赖
- 无新增依赖
